# AI Legal Search Yemen

منصة قانونية احترافية تحوّل المشروع إلى **محرك بحث ذكاء اصطناعي قانوني** بدلاً من موقع محتوى أو مدونة.

## ما الذي يقدمه النظام الآن
- **مربع بحث رئيسي كبير** لطرح الأسئلة القانونية بصياغة طبيعية.
- **محادثة قانونية محفوظة** شبيهة بتجربة ChatGPT مع حفظ كامل للمحادثات.
- **رفع ملف وسؤاله مباشرة**: PDF / Word / TXT / Markdown / JSON.
- **استخراج المواد القانونية** تلقائياً عندما يحتوي الملف على مواد مرقمة مثل `المادة (12)`.
- **عرض المصادر والمواد المستخدمة** داخل كل إجابة.
- **نواة RAG قانونية**: بحث داخل قاعدة البيانات الخاصة أولاً، ثم إنشاء إجابة قانونية واضحة ومهنية.
- **وضع AI اختياري** عبر مزود OpenAI-compatible إذا تمت إضافة مفتاح API لاحقاً.
- **تخزين محلي موثوق** عبر SQLite مع دعم اختياري لتخزين الملفات في Supabase Storage.

## التجربة المستهدفة
مثال عملي:
1. يرفع المستخدم قانون العمل اليمني.
2. يسأل: **ما حقوق العامل عند الفصل التعسفي؟**
3. النظام يبحث داخل النص.
4. يستخرج المادة الأقرب.
5. يقدّم شرحاً قانونياً واضحاً.
6. يذكر رقم المادة والمصدر المستخدم في الإجابة.

## التشغيل المحلي
```bash
npm install
npm run build
npm test
npm --workspace backend run start
```

- الواجهة: `http://localhost:5173` أثناء التطوير أو `http://localhost:8080` عبر Docker/Nginx.
- الـ API: `http://localhost:4000`
- Swagger: `http://localhost:4000/docs`

## متغيرات البيئة
انسخ `.env.example` إلى `.env` وعدّل القيم حسب بيئة التشغيل.

## الهيكل
- `frontend/` واجهة المستخدم القانونية.
- `backend/` محرك البحث القانوني ورفع الملفات والمحادثات.
- `docs/` ملاحظات معمارية وتشغيلية.
- `nginx/` إعدادات البروكسي.

## ملاحظات معمارية
- التخزين الأساسي للبيانات والمحادثات: SQLite.
- تخزين الملفات: محلي أو Supabase Storage.
- توليد الإجابات:
  - **بدون مفتاح AI**: إجابة قانونية استخراجية مهنية مع المواد والمصادر.
  - **مع مفتاح AI**: إجابة أذكى بصياغة لغوية أعلى مع الالتزام بالنصوص المسترجعة.
